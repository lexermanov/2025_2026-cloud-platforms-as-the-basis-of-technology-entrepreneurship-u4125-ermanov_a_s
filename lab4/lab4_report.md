## Отчет по лабораторной работе "Разработка инфраструктуры MVP AI приложения"  
**Цель работы:** 
Создать прототип AI-приложения в Google Cloud с проектированием инфраструктуры и оценкой экономической модели. 

# 1. Введение

В рамках лабораторной работы была выполнена разработка концепции AI-приложения с использованием облачной платформы Google Cloud Platform (GCP). Основная задача состояла в проектировании инфраструктуры, которая сможет масштабироваться по мере роста нагрузки на сервис.

При проектировании учитывались:

- минимизация затрат на начальном этапе;
- возможность быстрого масштабирования;
- отказоустойчивость;
- удобство сопровождения;
- готовность инфраструктуры к переходу в production.

Для реализации проекта были рассмотрены три состояния приложения:

1. Начальный этап (MVP);
2. Этап тестирования партнёрами;
3. Production-решение.

# 2. Описание AI-приложения

## Название приложения

AI Wardrobe Platform

## Назначение приложения

Приложение представляет собой AI-сервис, который позволяет пользователям взаимодействовать с ИИ-стилистом через веб-интерфейс. Система принимает пользовательские запросы, обрабатывает их при помощи AI-модели и возвращает результат в режиме реального времени.

## Основной функционал

- регистрация и авторизация пользователей;
- отправка запросов AI-модели;
- хранение истории запросов;
- аналитика использования;
- REST API для интеграций;
- административная панель.

## Предполагаемая нагрузка

| Этап | Количество пользователей | Одновременные запросы |
|---|---|---|
| MVP | 50–100 | до 10 |
| Партнёрское тестирование | 1000–3000 | до 100 |
| Production | 10000+ | 500+ |

# 4. Этап 1 — MVP (начальное состояние)

## Схема инфраструктуры

```text
Пользователь
    ↓
Firebase Hosting
    ↓
Cloud Run (Backend API)
    ↓
Vertex AI API
    ↓
Firestore / Cloud Storage
```

## Цель этапа

На первом этапе основной задачей является быстрое создание работающего прототипа с минимальными затратами.

## Используемые сервисы Google Cloud

| Сервис | Назначение |
|---|---|
| Cloud Run | Размещение backend и AI API |
| Firebase Hosting | Размещение frontend |
| Firestore | Хранение пользовательских данных |
| Cloud Storage | Хранение файлов |
| Vertex AI API | Использование AI-модели |
| Cloud Logging | Логирование |

## Экономическая модель MVP

| Компонент | Примерная стоимость в месяц |
|---|---|
| Cloud Run | 10–20 USD |
| Firestore | 5–15 USD |
| Cloud Storage | 2–5 USD |
| Vertex AI API | 20–50 USD |
| Firebase Hosting | 0–5 USD |
| Итого | ~40–95 USD |

# 5. Этап 2 — Тестирование партнёрами

## Схема инфраструктуры

```text
Пользователь
    ↓
Load Balancer
    ↓
GKE Autopilot
    ↓
Redis (Memorystore)
    ↓
Cloud SQL
    ↓
Vertex AI
```

## Изменения инфраструктуры

| Компонент | Изменение |
|---|---|
| Backend | Переход на GKE Autopilot |
| База данных | Переход на Cloud SQL |

| Балансировка | Подключение Load Balancer |
| Мониторинг | Cloud Monitoring |

## Экономическая модель этапа тестирования

| Компонент | Примерная стоимость в месяц |

| GKE Autopilot | 80–150 USD |
| Cloud SQL | 50–120 USD |
| Memorystore | 35–70 USD |
| Load Balancer | 20–40 USD |
| Cloud Storage | 10–20 USD |
| Vertex AI API | 150–300 USD |
| Monitoring & Logging | 20–40 USD |
| Итого | ~365–740 USD |

# 6. Этап 3 — Production

## Схема инфраструктуры

```text
Пользователь
    ↓
Cloud CDN
    ↓
Global Load Balancer
    ↓
GKE Cluster
    ↓
Redis Cluster
    ↓
Cloud SQL HA
    ↓
Vertex AI GPU
```

## Архитектура production-решения

| Компонент | Используемое решение |
|---|---|
| Контейнерная платформа | GKE Standard |
| AI-сервис | Vertex AI + выделенные GPU |
| База данных | Cloud SQL HA |
| Кэширование | Memorystore Redis Cluster |
| CDN | Cloud CDN |
| Балансировка | Global Load Balancer |
| Безопасность | IAM + Secret Manager + Cloud Armor |
| Мониторинг | Cloud Monitoring + Error Reporting |

## Экономическая модель production

| Компонент | Примерная стоимость в месяц |
|---|---|
| GKE Standard | 300–800 USD |
| GPU для AI | 500–3000 USD |
| Cloud SQL HA | 200–500 USD |
| Redis Cluster | 150–300 USD |
| Load Balancer + CDN | 80–200 USD |
| Monitoring & Logging | 50–150 USD |
| Cloud Storage | 30–100 USD |
| Vertex AI | 500–2000 USD |
| Итого | ~1810–7050 USD |

# 7. Обоснование выбора решений

Использование serverless-сервисов на этапе MVP позволяет существенно сократить расходы и ускорить разработку продукта. Однако при росте нагрузки такие решения становятся менее эффективными с точки зрения стоимости и контроля над инфраструктурой.

Переход на Kubernetes на втором этапе обеспечивает:

- более эффективное использование ресурсов;
- масштабируемость;
- устойчивость к нагрузке.

Production-инфраструктура ориентирована на:

- высокую доступность;
- безопасность;
- отказоустойчивость;
- возможность масштабирования.
